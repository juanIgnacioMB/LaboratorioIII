  • Cuales son los clientes http y los servidores http mas utilizados?

    • Los clientes mas usados son los buscadores web y los servidores HTTP el Apache y Microsoft IIS

  • Que verbos admite un comando en el requerimiento HTTP?

    • GET (solo de lectura), POST (Utilizado normalmente para actualizar recursos en un servidor), DELETE (Utilizado para borrar recursos en un servidor), PUT (Utilizado para crear recursos en un servidor)

  • Que contenido lleva el body de un requerimiento HTTP?

    • Envia datos a un servidor, como los datos de un formulario o archivos para realizar un upload

  • Que diferencia existe entre un URL y un URI?

    • La URL es la direccion web de un recurso de Internet en cambio la URI ubica el lugar donde se encuentra

  • Como se almacena informacion relacionada con las respuestas http?

    • Lo hace a traves de cookies. Las cookies (a menudo conocidas como cookies de Internet) son archivos de texto con pequenos datos, que se utilizan para identificar tu ordenador cuando utilizas una red

  • Porque el HTTP se considera STATE LESS?

    • Por que cada solicitud que el cliente hace es una transaccion independiente, que no esta relacionada a ninguna solicitud previa y cada una es diferente para el servidor

  • Como va a mejorar la velocidad de la WEB en poco tiempo

    • A traves del protocolo HTTP/2, lo hace compactando cabeceras, prioritizando elementos a transferir y multiplexando sin esperar respuestas. Ademas de trabajar con paquetes binarios en vez de paquetes en formato de ASCII

  • Como va a mejorar el comportamiento de una aplicacion WEB en ambientes ruidosos o de debil conexion?

    • Con la llegada del HTTP/3 que utiliza el protocolo QUIC, el cual utiliza UDP en vez de TCP, que produce conexiones mas cortas, eficientes y de facil establecimiento, lo que hace que la reconeccion sea mas facil en ambientes de mucho ruido o debil conexion.
